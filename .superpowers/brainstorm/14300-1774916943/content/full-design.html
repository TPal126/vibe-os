vibe-os UI Enhancement: Full Design

RStudio-inspired quadrant layout with live architecture, context controls, and independent multi-session support

Design Philosophy

Make the invisible visible. Vibe-os doesn't replace Claude CLI — it makes the working context of your AI sessions visible and controllable. Like RStudio showed R developers their environment, packages, and plots alongside the console, vibe-os shows developers their architecture, context layer, and activity alongside the chat.

Principle 1

Everything visible simultaneously — no hidden panels hiding value

Principle 2

Context is controllable — toggle skills, repos, and budgets in real-time

Principle 3

Sessions are independent worlds — switching tabs reorients the full cockpit

3. Architecture Diagram — Zoom Levels

Level 1 — Repo Topology (default)
  • One box per repo/service
  • Lines show: imports, API calls, shared deps
  • Nodes pulse when active session is touching them
  • Color-coded borders: blue (active), green (recently changed), gray (idle)
  • Quick stats: framework, route/component count
Level 2 — Module Detail (click to drill)
  • Shows internal directory/module structure
  • Dependency arrows between modules
  • Test status per module (pass/fail counts)
  • Recently changed files highlighted
  • Breadcrumb navigation back to Level 1

Real-time updates: Architecture diagram re-renders as Claude makes changes. When a file is edited, its module node briefly flashes. When a new dependency is added, the arrow animates in. Data comes from the existing SurrealDB graph indexer, extended with a file-watcher trigger.

4. Independent Sessions

Each session is an independent world with its own:

✦ Active repo set
✦ Skill selections (checkboxes)
✦ Token budget & cost tracking
✦ Architecture view (filtered)
✦ Audit trail & decision history
✦ Chat conversation

Switching session tabs swaps all four quadrants to that session's state. The cockpit reorients completely. Power users can pin a quadrant (right-click → "Pin pane") to keep it visible across session switches.

5. Interactions & Shortcuts

Drag dividers — resize any quadrant
Double-click tab bar — maximize pane (Esc to restore)
Click architecture node — drill into module view
Checkbox skills — toggle context, see token impact
Switch session tab — all quadrants update together
Right-click pane → Pin — persist across session switches
Ctrl+1/2/3/4 — focus quadrant by number
Ctrl+Shift+C — editor overlay (existing)
