Architecture Pane: What Should the Live Diagram Show?

This is the top-right quadrant — the "what does my system look like" view. It updates in real-time as Claude works.

A — Repo & Service Topology

High-level boxes for each repo/service. Shows how they connect (imports, API calls, shared types). Nodes glow when the active session is touching them. Compact, always useful, easy to parse at a glance.

B — Module-Level Detail

Zoomed into one repo at a time. Shows internal modules, their relationships, test status, and what was recently changed. More detail but limited to one repo — you'd switch repos via dropdown or by clicking a node in the repo-level view.

Recommendation: Both — as Zoom Levels

Default to A (Repo Topology) as the overview. Click any repo node to drill into B (Module Detail) for that repo. Breadcrumb navigation to go back up. This mirrors how maps work — zoom in for detail, zoom out for context.

The architecture pane always starts at the repo-level view. As Claude works, changed nodes pulse. Click to drill down. Press Escape or click the breadcrumb to zoom back out.
